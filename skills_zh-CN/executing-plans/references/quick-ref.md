# 快速参考

本文档提供执行计划技能的关键原则速查和质量检查清单。

---

## 关键原则

| 原则           | 说明                                     |
| -------------- | ---------------------------------------- |
| 强制更新       | **执行状态文件更新是不可跳过的强制要求** |
| 批判性审阅     | 永远不要跳过阶段一                       |
| 阶段声明       | 每轮对话开始时声明当前阶段               |
| 遇到阻碍即停止 | 询问比猜测更好                           |
| 精确遵循       | 不要即兴发挥计划步骤                     |
| 频繁提交       | 如果指定，在每个工作步骤后提交           |

---

## 注意事项

1. **始终在开始时声明技能使用**
2. **阶段声明是强制性的** - 每轮对话开始时声明当前阶段
3. **批判性审阅是强制性的** - 永远不要跳过阶段一
4. **强制更新** - 执行状态文件更新是不可跳过的强制要求
5. **遇到阻碍即停止** - 询问比猜测更好
6. **精确遵循** - 不要即兴发挥计划步骤
7. **报告偏差** - 如果必须偏离，记录原因到执行状态文件
8. **频繁提交** - 如果指定，在每个工作步骤后提交
9. **未经同意不在主分支操作** - 需要明确批准

---

## 阶段声明格式

每轮对话开始时声明：

```
【执行阶段 X/3】正在执行：[阶段名称]
当前处理：[子系统名称]（如适用）
```

**阶段定义**：

- 阶段 1/3：加载并审阅计划
- 阶段 2/3：执行任务
- 阶段 3/3：完成与报告

---

## 审阅检查清单

执行前，批判性评估计划：

- [ ] 目标是否清晰且可实现？
- [ ] 所有文件路径是否精确且正确？
- [ ] 代码片段是否完整（非占位符）？
- [ ] 测试命令对项目是否准确？
- [ ] 每个步骤是否足够细粒度（2-5分钟）？
- [ ] 任务之间的依赖关系是否清晰？
- [ ] 技术栈是否适当？

---

## 质量检查清单

### 执行前

- [ ] 计划文件已定位且可读
- [ ] 批判性审阅已完成
- [ ] 疑虑已解决（如有）
- [ ] 已从计划创建待办列表
- [ ] 执行状态文件已创建
- [ ] 不在主分支上（或已获得明确批准）

### 执行中

- [ ] 每个步骤完全按书面执行
- [ ] 验证已运行并通过
- [ ] 任务列表实时更新
- [ ] 执行状态文件实时更新
- [ ] 按指定进行提交

### 执行后

- [ ] 所有任务标记为已完成
- [ ] 执行状态文件已更新为完成状态
- [ ] 最终验证通过
- [ ] 完成报告已生成
- [ ] 无未报告的偏差

---

## 任务状态

| 状态   | 说明                     |
| ------ | ------------------------ |
| 已完成 | 任务已完成，所有验证通过 |
| 进行中 | 任务正在执行中           |
| 待执行 | 任务尚未开始             |
| 阻塞   | 任务因问题被阻塞         |

---

## 恢复执行流程

1. 读取执行状态文件，恢复上下文
2. 读取计划文档，扫描任务标记状态
3. 识别中断位置
4. 执行代码验证：
   - 检查 Git 状态
   - 检查已完成任务的代码是否存在
   - 运行相关测试验证已完成任务正常
5. 向用户报告恢复选项
6. 根据用户选择执行相应操作

---

## 停止条件

**在以下情况下立即停止执行**：

| 情况             | 行动                   |
| ---------------- | ---------------------- |
| 缺少依赖项       | 请求用户安装或提供     |
| 测试反复失败     | 调试并在卡住时寻求指导 |
| 指令不明确       | 请求澄清，不要猜测     |
| 计划存在关键缺陷 | 返回阶段一进行计划修订 |
| 出现意外错误     | 记录错误并寻求帮助     |

**不要强行突破阻碍** - 停止并询问

---

## 重新审视条件

**在以下情况下返回计划审阅**：

- 用户根据反馈更新了计划
- 基本方法需要重新思考
- 多个任务被同一问题阻碍
- 尽管遵循计划，验证仍持续失败
