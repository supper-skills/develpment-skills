## 项目类型识别

| 类型     | 特征                     | 典型技术栈                         |
| -------- | ------------------------ | ---------------------------------- |
| Web 应用 | 浏览器访问、前后端分离   | React/Vue/Next.js + Node/Python/Go |
| CLI 工具 | 命令行交互、脚本自动化   | Node.js/Python/Rust/Go             |
| 库/SDK   | 被其他项目引用、API 设计 | TypeScript/Python/Rust             |
| 微服务   | 独立部署、服务间通信     | Docker/Kubernetes + 任意语言       |
| 桌面应用 | 本地运行、GUI 界面       | Electron/Tauri/Qt                  |

### 确认事项

### 常见目录结构

**Web 应用（前端）**：

```
project/
├── src/
│   ├── components/    # 组件
│   ├── hooks/         # 自定义 hooks
│   ├── pages/         # 页面
│   ├── services/      # API 服务
│   ├── utils/         # 工具函数
│   └── types/         # 类型定义
├── tests/
├── public/
└── package.json
```

**CLI 工具**：

```
project/
├── src/
│   ├── commands/      # 命令处理
│   ├── utils/         # 工具函数
│   └── index.ts       # 入口
├── tests/
└── package.json
```

**库/SDK**：

```
project/
├── src/
│   ├── index.ts       # 导出入口
│   └── modules/       # 功能模块
├── tests/
├── examples/          # 使用示例
└── package.json
```

## 基础设施决策

### 版本控制策略

| 策略        | 适用场景             | 特点               |
| ----------- | -------------------- | ------------------ |
| Git Flow    | 大型项目、多版本并行 | 分支多、流程严格   |
| GitHub Flow | 持续部署、小团队     | 简单、主分支可部署 |
| Trunk Based | 高频发布、CI/CD 成熟 | 单主分支、特性开关 |

### CI/CD 需求评估

- [ ] 是否需要自动化测试？
- [ ] 是否需要自动化构建？
- [ ] 是否需要自动化部署？
- [ ] 部署目标是什么环境？

### 容器化需求

- [ ] 是否需要 Docker 支持？
- [ ] 是否需要 Kubernetes 编排？
- [ ] 开发环境是否需要容器化？

## 关键问题清单

在项目初始化场景中，必须向用户确认以下问题：

### 1. 目标用户

> "这个项目的主要目标用户是谁？"

- 内部团队使用
- 外部开发者
- 终端用户
- 其他

### 2. 项目规模

> "预期的项目规模和复杂度？"

- 小型（1-2 人月）
- 中型（3-6 人月）
- 大型（6 人月以上）

### 3. 环境支持

> "是否需要支持多环境（开发、测试、生产）？"

- 仅开发环境
- 开发 + 生产
- 开发 + 测试 + 生产

### 4. 合规与安全

> "是否有合规性或安全性要求？"

- 无特殊要求
- 数据安全要求
- 行业合规要求（如 GDPR、HIPAA）
- 其他
