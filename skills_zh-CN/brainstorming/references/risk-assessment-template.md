# 风险评估模板

本文档提供设计阶段的风险识别和评估框架，供头脑风暴阶段按需查阅。

---

## 风险类别

| 类别 | 示例风险 | 典型表现 |
|------|----------|----------|
| 技术风险 | 技术方案未经验证、依赖项不稳定、性能瓶颈 | 技术选型失误、第三方服务不可用 |
| 范围风险 | 需求蔓延、功能边界模糊、依赖外部系统 | 需求不断变化、项目延期 |
| 资源风险 | 人力不足、时间紧迫、技能缺口 | 团队成员变动、工期压缩 |
| 集成风险 | 第三方 API 变更、数据迁移复杂、兼容性问题 | 外部依赖不稳定、接口变更 |
| 安全风险 | 数据泄露、认证漏洞、权限控制缺陷 | 安全漏洞、数据丢失 |

---

## 风险评估矩阵

```
                影响程度
              低      中      高
         ┌────────┬────────┬────────┐
    高   │ 监控   │ 缓解   │ 优先   │
发       ├────────┼────────┼────────┤
生       中   │ 接受   │ 监控   │ 缓解   │
概       ├────────┼────────┼────────┤
率       低   │ 接受   │ 接受   │ 监控   │
         └────────┴────────┴────────┘
```

---

## 风险评估流程

```
1. 识别风险
   └── 列出所有可能影响项目的风险因素

2. 评估影响
   └── 按严重程度分级：高/中/低

3. 评估概率
   └── 估计发生可能性：高/中/低

4. 制定缓解策略
   └── 针对高影响/高概率风险制定应对方案

5. 记录风险清单
   └── 在设计文档中包含风险章节
```

## 风险记录模板

在规格文档中记录风险评估：

```markdown
## 风险评估

| 风险描述 | 类别 | 影响 | 概率 | 缓解策略 |
|----------|------|------|------|----------|
| 第三方API可能变更 | 集成 | 高 | 中 | 准备备选方案，封装调用层 |
| 团队成员可能变动 | 资源 | 中 | 中 | 文档化关键知识，交叉培训 |
| 性能可能不达标 | 技术 | 高 | 低 | 早期性能测试，预留优化时间 |
```

---

## 缓解策略示例

| 策略类型 | 适用场景 | 示例 |
|----------|----------|------|
| 技术验证 | 技术方案不确定 | 对不确定的技术方案进行原型验证 |
| 增量交付 | 功能复杂度高 | 将大功能拆分为可独立交付的小功能 |
| 备选方案 | 关键依赖风险 | 为关键依赖准备替代方案 |
| 监控预警 | 运行时风险 | 建立关键指标的监控机制 |
| 文档化 | 知识风险 | 记录关键决策和实现细节 |
