# 质量检查清单

在阶段 6 用户确认计划前，使用此清单进行最终检查。

## 目录

- [计划完整性](#计划完整性)

- [纲领文件验证](#纲领文件验证)

---

## 计划完整性

- [ ] 计划标题完整，包含目标、架构、技术栈
- [ ] 文件结构在任务前已记录
- [ ] 所有文件路径精确明确
- [ ] 所有代码片段完整（非占位符）
- [ ] 所有命令精确且有预期输出

---

## 任务质量

- [ ] 每个任务有清晰的文件引用（创建/修改/测试）
- [ ] 每个步骤粒度适中（2-5 分钟）
- [ ] 测试先于实现
- [ ] 每个任务独立可测试

---

## 分块质量

- [ ] 每块 300-500 行且逻辑自洽
- [ ] 块间依赖明确
- [ ] 没有孤立引用其他块

---

## 范围验证

- [ ] 计划覆盖单一内聚功能/子系统
- [ ] 如有多个子系统，纲领文件已完整填写
- [ ] 子系统间依赖关系已记录

---

## 纲领文件验证

- [ ] 纲领文件已创建并更新
- [ ] 会话状态信息完整
- [ ] 决策记录完整
- [ ] 变更历史已更新（如有变更）
- [ ] 所有子系统进度已记录（如适用）

---

## 评审验证

- [ ] 所有块已通过块级评审
- [ ] 所有子系统已通过子系统评审（如适用）
- [ ] 最终评审已完成

---

## 关键原则速查

| 原则 | 说明 |
|------|------|
| 纲领统一 | 使用纲领文件统一管理会话状态、决策记录、项目信息 |
| 决策可追溯 | 所有重要决策记录在案，防止重复讨论 |
| 变更可控 | 变更经过影响分析和评审，确保一致性 |
| 模式复用 | 常见场景使用预定义模式，提高效率 |
| 多级评审 | 块级、子系统级、最终评审确保质量 |
| 阶段声明 | 每轮对话声明当前阶段 |
| 分块输出 | 每块 300-500 行，逻辑自洽 |
| 逐个处理 | 多子系统时逐个处理，完成后更新纲领 |
| 精确完整 | 精确路径、完整代码、精确命令 |
| 测试优先 | 每个实现步骤跟随测试 |
| 语言适配 | 所有示例使用项目编程语言 |
